Git 基本使用

  1. git init //初始化仓库
  2. git add .(文件name) //添加文件到本地仓库
  3. git commit -m "first commit" //添加文件描述信息
  4. git remote add origin + 远程仓库地址 //链接远程仓库，创建主分支
  5. git pull origin master // 把本地仓库的变化连接到远程仓库主分支
  6. git push -u origin master //把本地仓库的文件推送到远程仓库

Problem

Descrition:Updates were rejected because the tip of your current branch is behind

Solution

  • push前先将远程repository修改pull下来
    1. $git pull origin master
    2. $git push -u origin master
  • 使用强制push的方法：
    1. $ git push -u origin master -f

    这样会使远程修改丢失，一般是不可取的，尤其是多人协作开发的时候

  • 若不想merge远程和本地修改，可以先创建新的分支：
    1. $ git branch [name]
    2. $ git push -u origin [name]

添加新文件（修改）后的二次提交

  1. git add 文件名
  2. git add * 提交所有文件
  3. git commit -m "这一次提交的描述,这一步不能省略"
  4. git status "查看当前状态，可以省略"
  5. git push -u origin note "note 是你要提交到的那个branch"

怎么在本地电脑建立github链接

  1. 在GitHub上新建一个respositories
  2. 生成ssh密匙 ssh-keygen -t rsa –C "你github邮箱"
  3. windows电脑查找文件: 我的电脑 => 用户 => 用户名文件夹 => .ssh(隐藏文件) => id_rsa.pub(存有密匙)
  4. github账户配置密匙 //在网页端GitHub
  5. 暂存区 => 远程仓库 git remote add origin https://github.com/BullFei/testDemo.git git push -u origin master
